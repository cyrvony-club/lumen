# Модель данных Lumen

## Метаданные (обязательные для опубликованных объектов)

| Поле | Описание |
|------|-----------|
| `id` | Стабильный идентификатор (строка, не меняется при мелких правках) |
| `type` | Тип сущности Lumen или тип артефакта |
| `title` или `name` | Человекочитаемое имя |
| `summary` | Краткая выжимка для ranking/filtering |
| `tags` | Массив тегов |
| `updated_at` | UTC ISO-8601 |
| `version` | Версия объекта (semver или договорённость проекта) |
| `canonical_url` | Канонический URL опубликованного объекта |

Опционально (рекомендуется blueprint): `language`, `related_ids`, `entity_ids`, `relations`, `source_of_truth`, `deprecation_status`.

## Сущности домена

| Сущность | Назначение |
|----------|------------|
| `Project` | Контейнер: настройки, связанные брифы, пайплайны, стиль |
| `ContentBrief` | Входное ТЗ на генерацию: цели, ограничения, ссылки на аудиторию/канал/гайд |
| `Source` | Исследовательский источник (URL, выдержка, доверие, дата обращения) |
| `ContentTemplate` | Структура выходного контента (секции, поля, форматы) |
| `Prompt` | Версионируемый текст/шаблон промпта с параметрами |
| `GenerationPipeline` | Связка шаблон + промпты + модельные параметры + постобработка |
| `GeneratedArtifact` | Результат генерации (текст/markdown, метаданные, статус жизненного цикла) |
| `StyleGuide` | Правила тона, терминов, запретов |
| `Audience` | Сегмент читателя/потребителя retrieval |
| `Channel` | Канал публикации (blog, docs, feed-only, API-only, …) |
| `Review` | Запись ревью артефакта (автор роли, вердикт, комментарии) |
| `Revision` | Версия правок текста артефакта после ревью |
| `AgentRole` | Перечисление ролей агента (`curator`, `scout`, `architect`, `worker`) для аудита |

## Жизненный цикл `GeneratedArtifact`

Статусы (строгая последовательность по смыслу; допускаются повторы `revision`):

1. `draft` — черновик до/после первичной генерации.
2. `generated` — финализированный вывод модели до человеческого/авто-ревью.
3. `reviewed` — есть хотя бы одна запись `Review`.
4. `approved` — готово к публикации в knowledge surface.
5. `published` — доступно через API/фиды/exports.
6. `archived` — снято с активной выдачи, история сохранена.

## Отношения (минимум)

Рекомендуемые типы связей между сущностями (для графа и расширения retrieval): `part_of`, `depends_on`, `used_for`, `mentions`, `updated_by`, `related_to` (см. blueprint).

## Файлы схем

- `schema/entity.schema.json` — общая оболочка Lumen-сущности.
- `schema/chunk.schema.json` — чанк для RAG/bulk ingest.
