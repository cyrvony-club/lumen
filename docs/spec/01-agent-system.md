# Агентная система Lumen

## Роли и модели

| Роль | Модель | Функция |
|------|--------|---------|
| **Куратор (Curator)** | Composer 2 | Координация агентов, распределение задач и контекста, хранение **общего контекста** сессии/проекта |
| **Скаут (Scout)** | GPT-5.5 | Исследование источников (доки, web, внутренние репозитории по правилам), синтез выжимок и списков находок |
| **Архитектор (Architect)** | GPT-5.5 | Анализ выводов скаута, **принятие решений**, фиксация концепций, спецификации, критериев качества |
| **Рабочий (Worker)** | Composer 2 | Реализация по **явному** ТЗ; без самостоятельной смены архитектуры; при неясности — вопрос **архитектору** |

## Параллельные скауты

Если темы исследования **независимы** (разные домены, нет общего состояния), допускается до **трёх** скаутов одновременно. Куратор явно разводит контекст и ожидаемые артефакты по каждому потоку.

## Эскалация

1. **Worker → Architect**: неоднозначность требований, противоречие с blueprint, выбор между альтернативами данных/API.
2. **Любая роль → Curator**: приоритеты, смена scope, конфликт задач между потоками.

## Выходы по ролям (ожидаемые артефакты)

- **Скаут**: сводка источников, извлечённые факты, риски и пробелы, ссылки на `Source` (в терминах модели данных).
- **Архитектор**: решения по схеме данных, эндпоинтам, статусам артефактов, критериям валидации публикации.
- **Рабочий**: код, схемы, OpenAPI, JSONL/фиды — строго по утверждённой спецификации.
